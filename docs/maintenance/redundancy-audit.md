# Redou Redundancy Audit

Date: 2026-05-18

Scope: duplicate assets, duplicate scripts, and generated/runtime debris in the
Redou workspace. This report only records audit findings. No files were
deleted, moved, or reconfigured during this pass.

## Method

- Compared candidate files with SHA-256 hashes.
- Checked references with `rg` across package scripts, Electron source,
  PowerShell/shell scripts, docs, and build configuration.
- Checked generated paths against `.gitignore`, `git ls-files`, and
  `git check-ignore`.
- Treated `vendor/hermes` as a vendored upstream tree. Same-name files inside
  that tree are not automatically redundant.

### Asset Notes

- The root icon copies and desktop icon copies are byte-identical, but they
  currently serve different paths.
- `apps/desktop/assets/icons/redou-agent.ico` is used for the BrowserWindow icon
  through `desktopAssetPath("icons", "redou-agent.ico")`.
- `apps/desktop/assets/icons/redou-agent.ico` and
  `apps/desktop/assets/icons/redou-agent-install.ico` are used by
  electron-builder's `win.icon`, `nsis.installerIcon`, and
  `nsis.uninstallerIcon`.
- `assets/icons/redou-agent.ico` and
  `assets/icons/redou-agent-install.ico` are used for generated Windows
  shortcuts. Deleting them without changing the shortcut generator would break
  root shortcut creation.
- The duplicate banners are the only asset pair with no direct runtime or script
  reference found in this pass. Choose one canonical location before deleting.

1. Pick whether `assets/banner.png` or `apps/desktop/assets/banner.png` is the

   inside `vendor/hermes/scripts/`. If upstream sync should remain clean, keep
   the vendor copies.
3. For workspace cleanup, remove only ignored generated/runtime directories in a
   separate deletion pass, then immediately run the relevant install/build
   commands before runtime validation.
